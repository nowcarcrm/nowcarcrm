# NOWCAR CRM QA Checklist

## 1) 공통 접근
- `npm run dev` 실행 후 `http://localhost:3000` 접속
- 관리자 계정으로 로그인
- 우측 하단 전역 AI 버튼(🤖) 노출 확인

## 2) LeadDetailModal 저장 동작
- 경로: `신규`, `상담중`, `계약`, `인도완료` 목록에서 임의 고객 `상세` 진입
- 기본정보 탭:
  - `메모`, `계약기간`, `보증금/선납금(금액)` 수정 후 저장
  - 버튼이 `저장 중...`으로 바뀌고 비활성화되는지 확인
  - `기본정보 저장 완료` 토스트 확인
  - 저장 직후 `저장 반영 중...` 오버레이 표시 후 최신값 유지 확인
- 상담기록 탭:
  - 기록 추가 저장 시 `기록 저장 완료` 토스트 확인
  - 에러 시 에러 토스트 확인
- 견적 이력 탭:
  - 견적 추가 저장 시 `견적 저장 완료` 토스트 확인
  - 새로고침 후 견적 이력 유지 확인
- 계약 고객 탭:
  - 계약 저장 시 `계약 저장 완료` 토스트 확인
  - staff 계정에서도 저장 가능 여부 확인
- 출고 진행 탭:
  - 출고 저장 시 `출고 저장 완료` 토스트 확인

## 3) 관리자 직원 필터
- 경로: `전체 상담 고객`
- 사이드바 `담당 직원 필터`로 특정 직원 선택
- 목록이 해당 직원 고객만 보이는지 확인
- 브라우저 새로고침 후 필터 유지 확인
- URL `managerUserId` 파라미터 유지 확인

## 4) 날짜 필터
- 경로: `전체 상담 고객`
- 빠른 버튼 `오늘 / 이번달 / 전체` 각각 동작 확인
- 연/월/일 드롭다운과 조합 필터 확인
- 월별 배지(예: `2026-04 (N)`) 클릭 시 해당 월로 필터링되는지 확인

## 5) 인도완료 엑셀
- 경로: `인도완료 고객`
- `엑셀 다운로드` 버튼 클릭
- 파일 컬럼 헤더가 아래 순서로 나오는지 확인:
  - 고객명
  - 연락처
  - 담당자
  - 차량명
  - 계약일
  - 인도일
  - 보증금
  - 계약기간

## 6) 전역 AI 팝업 UI
- 모든 관리자 페이지에서 AI 버튼 노출 확인
- 모바일 뷰포트에서 팝업이 화면 하단 전체 폭으로 열리는지 확인
- LeadDetailModal 열림 상태에서 AI 패널이 모달 위로 침범하지 않는지 확인

## 7) 죽은 링크 점검
- 사이드바에 `출고` 메뉴가 없는지 확인
- `/leads/export-progress` 직접 접속 시 정상 라우트가 아님(의도된 제거) 확인
- 상담결과 변경 후 이동이 `계약`/`인도완료` 등 유효 경로로만 이동하는지 확인

## 8) 오류 로그 기준
- 브라우저 콘솔에서 저장/조회 시 불필요한 `console.log` 출력이 과도하지 않은지 확인
- 에러 상황에서만 `console.error`가 표시되는지 확인
